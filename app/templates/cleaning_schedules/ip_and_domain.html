TLNR:

We need either the administrator password or to spend money to access the website without wifi

Why?

Currently, we can only access the website with 12-digit number while connected to the house wifi, which is not that ideal. Here's why:

To access the server running locally, or any server a web service is running on, we need to know its ip address. The 12-digit number we are using is the IPV4 address.

Typically, we use domain names like minecraft.net or google.com instead of IP addresses. This is possible thanks to DNS servers. However, it is not free because there are only limited numbers of nice domains (e.g. apple.com and pear.com are both occupied) and where there resources are limited, there is a price to pay. Also, DNS servers cost money to run and maintain either.

Though there is a fix to the "domain vs ip" problem, which I will mention later in the page, there isn't really an easy fix to not being able to access it without connecting to the house wifi. While we do fortunately have both public ipv4 and ipv6 addresses, we also have a firewall at the router blocking all inbound connections from outside the house.

To allow connections to the local server, I need to modify the firewall settings, which requires the administrator password for the router. Since no one knows it, I can't. There are a few ways to work around it but they are either not free or have their own limitations

What alternatives do we have?

If you do not want to remember the 12 numbers, the simplest option is to let your browser remember the ip address so it can auto-complete the address for you.

You can also modify the "hosts" file to assign a customized name for the ip address, so you only need to type in the customized name next time you access the website.

As for accessing the website without connection to the house wifi, maybe you can try something like a vpn or a proxy, but they are probably not free.
